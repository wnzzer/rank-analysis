# UI 优化方案

为了提升软件的高级感与精致度，以下是按照优先级排序的优化建议。

## 1. 字体优化 (Typography) - **优先级：最高**
目前的界面使用系统默认字体，缺乏个性且在不同设备上表现不一。
*   **建议内容**：
    *   引入开源的现代无衬线字体（如 `Inter`）作为全局字体，提升阅读体验。
    *   针对**数字数据**（KDA、补刀、伤害），使用专门的字体（如 `Oswald`），让数据看起来更像“仪表盘”，增加电竞感。
    *   适当增加正文行高，避免文字过于密集。
*   **预期效果**：界面瞬间变得干净、利落，数据展示更具冲击力。

## 2. 色彩与背景重构 (Colors & Background) - **优先级：高**
目前 `App.vue` 使用了纯黑 (`black`) 背景，且部分边框使用了高对比度的 `#ccc`。
*   **建议内容**：
    *   **背景色**：放弃纯黑，改用**深空灰/蓝灰色**（例如 `#101014` 或 `#18181c`）。这比纯黑更有质感，且能减少视觉疲劳。
    *   **边框线**：将 `#ccc` 这种亮色边框改为**深色半透明**（例如 `rgba(255, 255, 255, 0.08)`）。高级感的边框应该是“似有若无”的。
    *   **主题色**：微调胜利/失败的红绿颜色，使其饱和度降低一点，或者更偏向荧光色（电竞风），避免“默认红绿”的廉价感。
*   **预期效果**：界面层次感增强，不再像是一个简单的网页，更像原生应用。

## 3. 间距与留白 (Spacing & Layout) - **优先级：中**
在 `RecordCard.vue` 中，您使用了 `padding: 2px` 和 `gap: 1px`，信息密度过高。
*   **建议内容**：
    *   **增加内边距**：将卡片的 Padding 至少增加到 `8px` 或 `12px`。
    *   **增加间距**：增加列表项之间的距离，让信息“呼吸”。
    *   **留白**：不要试图填满每一个像素，适当的留白能引导用户视线聚焦在重要数据上。
*   **预期效果**：界面不再拥挤，阅读战绩时更轻松，视觉压力减小。

## 4. 细节打磨 (Details: Radius & Shadows) - **优先级：低**
*   **建议内容**：
    *   **圆角**：统一使用较大的圆角（如 `8px` 或 `10px`），包括图片、卡片和按钮。
    *   **微阴影**：给卡片添加极淡的阴影或悬浮时的发光效果。
*   **预期效果**：增加界面的精致度和交互的愉悦感。
